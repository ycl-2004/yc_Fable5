# Fable 审计报告模板

> 完整报告仅在用户要求验证（「用 Fable 验证」）或 partner-skill 验收时产出。日常复杂交付用底部的 3 行尾注。

```markdown
# Fable 审计报告

- **审计对象**：<哪次任务/谁的产出（我自己 / Codex / 子代理）>
- **日期**：<审计日期>
- **总分**：<x/(计入维度×3)，y%>（N/A 维度不计入分母，全维计入时分母为 21；计分规则见 validation-rubric.md；分数只看趋势，判定以三档结论为准）
- **输入材料**：<拿到了什么：任务原文 / 交付物 / .fable/task.md / 过程记录；缺了什么>
- **归档**：<.fable/audits/YYYY-MM-DD-对象.md>

## 七维判定

| 维度 | 判定 | 证据 / 引用 |
|------|------|-------------|
| A 理解对齐 | PASS/FAIL/N/A | <具体引用> |
| B 路由合规 | PASS/FAIL/N/A | <具体引用> |
| C 范围与计划 | PASS/FAIL/N/A | <具体引用> |
| D 需求完备 | PASS/FAIL/N/A | <逐项对照结果> |
| E 验证证据 | PASS/FAIL/N/A | <证据层级评估> |
| F 沟通合规 | PASS/FAIL/N/A | <具体引用> |
| G 授权与安全 | PASS/FAIL/N/A | <高风险动作清点> |

## 结论：通过 / 有条件通过 / 不通过

<一句话理由；判定规则见 references/validation-rubric.md>

## 必须整改（FAIL 项，逐条给可执行动作）

1. <整改项：做什么就能过>

## 建议（不阻塞验收）

- <可选改进>
```

## 3 行自检尾注（日常复杂交付用）

```text
—— Fable 自检 ——
需求：x/x 项完成（追加 n 项已含）｜未尽：<无 / 具体项>
证据：<最强的 1-2 个：测试输出 / 文件路径 / 链接>
```
