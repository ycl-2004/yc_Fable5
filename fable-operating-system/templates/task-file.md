# Fable 任务文件模板

> 使用规则：超过约 5 步、跨会话、或委派给其他 agent 的任务，在项目目录建 `.fable/task.md`（可加入 .gitignore）。它有两个作用：①防上下文压缩丢状态 ②验证模式的第一证据源。需求清单只 append 不 replace；关键转折时更新决策日志。

```markdown
# Fable 任务文件

- **任务**：<一句话任务名>
- **日期**：<开始日期>
- **来源**：<用户原话或指令出处——保留原文，理解核对用>

## 目标（一句话）

<真实意图，不是字面要求>

## 验收标准

<可观察、可核对的完成判据，每条都要能回答"怎么证明">

## 需求清单（append-only）

- [ ] <原始需求项 1>
- [ ] <原始需求项 2>
- [ ] （追加 <日期>）<中途追加项——标注"追加"，不覆盖上面任何一项>

## 决策日志

- <日期> <决策> ：<理由>

## 验证证据

- <需求项> → <证据：测试输出/文件路径/链接/截图位置>

## 交接说明（可选）

<若任务会交给其他 agent 或下个会话继续：当前进度、下一步、注意事项>
```
