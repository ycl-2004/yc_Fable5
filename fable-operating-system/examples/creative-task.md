# 示例：创意设计任务（IP 配图）

## 任务

> 「用 YC 解释一下"上下文窗口"，做一张图。」

## 可观察的决策轨迹

### 1. 理解

- 表面："用 YC 解释 X + 做图" = yc-ip 的**显式触发句式**。
- 真实意图：一张可直接用于文章/社媒的成品配图，不是画面方案。
- 交付物：**图片文件**。yc-ip 的硬规则：必须真生成图，不能停在文字方案或 prompt——除非用户明说"只要 prompt"。
- 风格：用户没点名 Minimal/Rich → 按 Skill 默认走 Standard（纯白底、红发透明眼镜 chibi、手绘线条、中文短标注、中等信息密度）。
- 比例：没说用途 → 按 Skill 默认（文章 16:9）。若上下文能看出是社媒用途则 1:1。

### 2. 分类与路由

视觉 IP 类。命中 yc-ip（阻塞）→ 先加载 Skill 读其完整规则再动手。注意区分：如果用户说的是"小黑"或"文章插图/怪诞手绘"，应路由到 ian-xiaohei-illustrations，不是 yc-ip。

### 3. 规划（轻量档）

1. 加载 yc-ip，确认生图通道（默认 route）。
2. 设计画面：把"上下文窗口"翻译成一个视觉隐喻 + 少量中文标注。
3. 生成 → 真实查看 → 不合格就修 prompt 重生成。
4. 交付图片路径 + 说明创作选择。

### 4. 创意任务特有的沟通

- 产出后说明关键创作选择（选了什么隐喻、标注了哪几个词），让用户能**定向**提修改意见（"把标注改成 X"），而不是只能说"不满意重来"。
- 修改迭代时保持已确认的部分不动，只改被点名的部分。

### 5. 质量检查

- [ ] 图片文件真实存在且打开看过（构图、文字无乱码、IP 形象符合规范）
- [ ] 风格档位正确（默认 Standard，未被要求不用 Minimal/Rich）
- [ ] 中文标注拼写正确（生成图最常见的翻车点）
- [ ] 比例符合用途

## 本例的关键纪律

- 创意任务最大的失败模式是**交付方案而不是成品**——yc-ip 明确禁止停在文字层。
- 第二大失败模式是**不看结果就宣布完成**——生成图必须人眼级检查。
- 风格默认值来自 Skill 规则，不自由发挥；用户点名才切档。
